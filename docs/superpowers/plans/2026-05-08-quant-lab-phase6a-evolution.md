# Quant Lab Phase 6a — Autonomous Evolution & Re-calibration

> **For agentic workers:** Use superpowers:subagent-driven-development. Compact plan.

**Goal:** Make the live tournament self-improving without manual intervention. Three components:
1. **Weekly recalibration workflow** — every Sunday, re-run the walk-forward backtest with fresh data and commit new calibration. The meta-ensemble auto-picks up updated weights on Monday's run.
2. **Strategy lifecycle manager** — auto-pause strategies that consistently fail validation gates; auto-resume when their evidence recovers.
3. **Confidence-weighted live/backtest blend** — replace the binary live-overrides-backtest logic with a smooth ramp: live weight = min(1, days_of_live/365). New strategies trust backtest; mature strategies trust live evidence.

After this phase, the live system improves itself: every Sunday adds a week of fresh out-of-sample evidence to the calibration; every week the lifecycle pauses dead strategies; every day the ensemble weights smoothly track accumulated live evidence.

**API:**

```python
def confidence_weight(days_of_live: int, full_confidence_days: int = 365) -> float:
    """Map days of live history to [0, 1]. 0 days → 0 (full backtest). 365+ days → 1 (full live)."""

def blend_weights(
    backtest_weights: dict[str, float],
    live_weights: dict[str, float],
    days_of_live_per_bot: dict[str, int],
    full_confidence_days: int = 365,
) -> dict[str, float]:
    """For each bot in either dict, compute:
        alpha = confidence_weight(days_of_live_per_bot[bot])
        weight = (1 - alpha) * backtest + alpha * live
    Re-normalize to sum=1.0. Drop bots not in either dict."""
```

**Acceptance tests:**
- `confidence_weight(0) == 0.0`, `confidence_weight(365) == 1.0`, `confidence_weight(180) ≈ 0.5`
- Bot with 0 days live → uses backtest weight
- Bot with 365+ days live → uses live weight
- Bot with 180 days → ~50/50 blend
- Result re-normalizes to sum 1.0

```python
@dataclass
class LifecycleState:
    bot_id: str
    paused: bool
    paused_at: date | None
    pause_reason: str
    consecutive_fail_days: int
    consecutive_recovery_days: int

def evaluate_lifecycle(
    nav_history: dict[str, list[tuple[date, float]]],
    benchmark_returns: dict[str, list[float]],
    prior_state: dict[str, LifecycleState],
    today: date,
    *,
    fail_threshold_days: int = 90,
    recovery_threshold_days: int = 30,
    min_significance_for_active: float = 0.3,
    min_significance_for_resume: float = 0.5,
) -> dict[str, LifecycleState]:
    """For each bot in nav_history:
      - Compute trailing significance weight from live data (uses backtest/stats)
      - If active and bot has been below min_significance_for_active AND alpha < 0
        for fail_threshold_days consecutively → paused with reason
      - If paused and bot has been above min_significance_for_resume
        for recovery_threshold_days consecutively → resumed
      - Otherwise carry prior state forward, increment counters"""

def load_lifecycle_state(path: Path) -> dict[str, LifecycleState]: ...
def save_lifecycle_state(state: dict[str, LifecycleState], path: Path) -> None: ...
```

**Wire into `main.morning_command`:**

- Write `dashboard/data/lifecycle.json` for the validation page to render

**Acceptance tests:**
- Bot with 90 consecutive days of low significance + negative alpha → paused
- Bot with 30 days of strong recovery → resumed
- Bot with mixed history → counters reset properly
- State persists across calls (load → modify → save → load)

**Files:**

**Behavior:**
- Cron: `0 1 * * 1` (Monday 1 AM UTC = Sunday 8/9 PM ET, after the week's market close)
- Steps:
  1. Checkout repo
  2. Set up Python 3.11 + install deps
